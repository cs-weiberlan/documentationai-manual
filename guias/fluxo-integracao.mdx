## Visão geral

Este guia apresenta o caminho recomendado para integrar com a API Conta Simples, do primeiro contato até a operação em produção. Siga cada etapa na ordem para garantir uma integração segura e eficiente.

<Callout kind="info">
  **Tempo estimado:** A integração completa pode ser feita em poucos dias,
  dependendo da complexidade do seu caso de uso.
</Callout>

---

## Etapas da integração

<Steps>
  <Step title="Obter credenciais" titleType="p">
    Acesse o **Internet Banking da Conta Simples** para obter suas credenciais de **Sandbox**:

    [ib.contasimples.com/integracoes/api/credenciais](https://ib.contasimples.com/integracoes/api/credenciais)

    **O que você vai encontrar:**
    - **API Key** — identificador da sua aplicação
    - **API Secret** — segredo para autenticação

    <Callout kind="info">
    Não consegue ver as credenciais? O acesso depende do seu perfil de usuário e permissões. Solicite acesso ao responsável da sua empresa.
    </Callout>

    <Card title="Ambientes e credenciais" icon="lock" href="/comece-aqui/ambientes#como-acessar-suas-credenciais" horizontal={false}>
      Veja detalhes sobre como acessar e gerenciar credenciais.
    </Card>

  </Step>

  <Step title="Obter token de acesso" titleType="p">
    Com as credenciais em mãos, autentique-se via **OAuth 2.0 Client Credentials** para obter um `access_token`.

    ```bash
    # BASIC_BASE64 = "SUA_API_KEY:SEU_API_SECRET" encodado em base64
    curl --location 'https://api-sandbox.contasimples.com/oauth/v1/access-token' \
      --header "Authorization: Basic ${BASIC_BASE64}" \
      --header "Content-Type: application/x-www-form-urlencoded" \
      --data "grant_type=client_credentials"
    ```

    O token retornado tem validade de **30 minutos**. Implemente renovação automática.

    <Card title="Guia de Autenticação" icon="lock" href="/comece-aqui/autenticacao" horizontal={false}>
      Fluxo completo de autenticação, renovação e boas práticas.
    </Card>

  </Step>

  <Step title="Testar no Sandbox" titleType="p">
    Use o ambiente Sandbox para validar sua integração sem risco. Os dados são simulados e não têm impacto financeiro real.

    **Valide:**
    - Autenticação está funcionando
    - Requisições retornam dados esperados
    - Tratamento de erros está implementado

    <Card title="Ambientes" icon="flask" href="/comece-aqui/ambientes" horizontal={false}>
      Diferenças entre Sandbox e Produção.
    </Card>

  </Step>

  <Step title="Consultar transações de cartão" titleType="p">
    Faça sua primeira consulta ao extrato de transações usando o endpoint `POST /statements/v1/credit-card`. A resposta contém transações do período, incluindo dados do cartão, categoria, centro de custo e anexos.

    ```bash
    curl -X POST https://api-sandbox.contasimples.com/statements/v1/credit-card \
      -H "Authorization: Bearer {TOKEN}" \
      -H "Content-Type: application/json" \
      -d '{
        "startDate": "2025-01-01",
        "endDate": "2025-01-31",
        "limit": 10
      }'
    ```

    <Columns cols="2">
      <Card title="Endpoint: POST /statements/v1/credit-card" icon="code" href="/api-reference/introducao" horizontal={false}>
        Referência técnica completa com parâmetros e schemas.
      </Card>
      <Card title="Dicionário de Dados" icon="database" href="/referencia/dicionario-dados" horizontal={false}>
        Significado de cada campo retornado.
      </Card>
    </Columns>

  </Step>

  <Step title="Tratar paginação" titleType="p">
    Para grandes volumes de dados, a API retorna resultados paginados via cursor (`nextPageStartKey`).

    **Fluxo:**
    1. Envie a primeira requisição sem `nextPageStartKey`
    2. Se a resposta incluir `nextPageStartKey`, envie-o na próxima requisição
    3. Repita até `nextPageStartKey` ser `null` ou ausente

    <Card title="Boas Práticas — Paginação" icon="arrow-right" href="/guias/boas-praticas#paginação" horizontal={false}>
      Exemplo completo de paginação com código.
    </Card>

  </Step>

  <Step title="Consumir anexos" titleType="p">
    Transações podem incluir comprovantes (notas fiscais, recibos). Baixe-os via `GET /attachments/v1/content/{attachmentId}`.

    **Fluxo:**
    1. Consulte transações e verifique o array `attachments`
    2. Para cada anexo, extraia o `id`
    3. Faça `GET /attachments/v1/content/{id}` para baixar o arquivo binário
    4. Use o header `Content-Type` da resposta para determinar o formato (PNG, JPEG, PDF)

    <Columns cols="2">
      <Card title="Boas Práticas — Anexos" icon="shield" href="/guias/boas-praticas#download-de-anexos" horizontal={false}>
        Exemplo de download e processamento.
      </Card>
      <Card title="Endpoint: GET /attachments/v1/content/{id}" icon="code" href="/api-reference/introducao" horizontal={false}>
        Referência técnica do endpoint.
      </Card>
    </Columns>

  </Step>

  <Step title="Ir para produção" titleType="p">
    Quando sua integração estiver validada no Sandbox, solicite a promoção para o ambiente de **Produção**.

    **Checklist de go-live:**

    - Integração Sandbox validada com todos os fluxos
    - Autenticação com renovação automática de token
    - Paginação implementada corretamente
    - Tratamento de erros (401, 400, 500) implementado
    - Lógica de retry com backoff exponencial
    - Idempotência em operações críticas
    - Credenciais armazenadas em cofre de segredos
    - Monitoramento e alertas configurados
    - Logging estruturado com request_id

    <Card title="Ir para Produção" icon="rocket" href="/comece-aqui/ambientes#promoção-para-produção" horizontal={false}>
      Veja os requisitos e o checklist de go-live.
    </Card>

  </Step>
</Steps>
